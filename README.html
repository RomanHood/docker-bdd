Behavior-Driven Docker Development

Pre-Requisites - Docker & Fig

Running Docker natively

This is the preferred approach. If you're running linux natively then just install Docker and Fig natively.

Running Docker on Boot2Docker (OSX)

Follow the instructions to install Boot2Docker and then install Fig

Running via Vagrant and Virtualbox

If you're on Windows, or can't get docker installed on your platform, we've provided a Vagrant setup that you can use. Install Virtualbox >= 4.3.20 as well as Vagrant >= 1.6.5 for your platform. Then get up and running:

$ vagrant up
$ vagrant provision

Once up and running, you'll run the fig commands from inside the vagrant image:

$ vagrant ssh

Testing Dependencies

In addition to installing docker and fig, you'll need to have Cucumber installed in order to run the tests.

  • install ruby (e.g. via rvm or a system package)
  • install cucumber gem install cucumber

Running the Tests

The tests ensure that the fig cluster is running, so running them is as simple as running cucumber. You can also just run a specific test:

$ cucumber features/web.feature

Starting the cluster

Running the tests also accomplishes this, but if you're too lazy to run the tests: $ fig run -d

When you make a change and want to rebuild a docker container

Again, running the tests accomplishes this, but if you want to do it manually: $ fig build
